# Phase 1: Foundation Plan

## Goal
Scaffold the base repositories, configure the database connection, create the core SQLAlchemy models, and implement the JWT-based authentication system with Role-Based Access Control.

## Context
This is Day 1 of our 3-Day Sprint. We need a solid backend foundation and a Next.js frontend scaffold to build upon for tomorrow's complex booking logic.

## Wave 1: Scaffolding & Setup
1. **Frontend Setup**: Scaffold Next.js app in `/frontend` directory using `create-next-app`.
2. **Backend Setup**: Scaffold FastAPI app structure in `/backend` directory.
3. **Dependencies**: Create `backend/requirements.txt` (FastAPI, Uvicorn, SQLAlchemy, psycopg2-binary, passlib, python-jose, pydantic, pydantic-settings, asyncpg).
4. **Database Connection**: Create `backend/app/database.py` with SQLAlchemy async engine and Neon PostgreSQL URL setup.

## Wave 2: Models & Auth Core
1. **SQLAlchemy Models**: Create `backend/app/models.py`.
   - `User` (id, name, email, password_hash, role, phone)
   - `Doctor` (id, user_id, specialization, experience, working_hours, slot_duration)
   - `Patient` (id, user_id, age, gender, blood_group)
   - `Appointment` (id, patient_id, doctor_id, date, start_time, end_time, status, symptoms, urgency, etc.)
2. **JWT Utilities**: Create `backend/app/auth.py` for password hashing (passlib) and token generation/decoding (python-jose).
3. **Auth Endpoints**: Create `backend/app/api/auth_routes.py` (Signup, Login, Token generation).

## Wave 3: CRUD Endpoints
1. **Admin APIs**: `backend/app/api/admin_routes.py` (Manage doctors - Add, Edit, Delete).
2. **Doctor/Patient Profiles**: Endpoints to fetch their respective profile information based on JWT tokens.

## Verification
- Can a user sign up as a Doctor or Patient?
- Can they log in and receive a valid JWT?
- Are the database tables properly created when the backend starts?
