# Phase 3: Frontend Delivery Plan

## Goal
Build a modern, highly aesthetic, and responsive Next.js frontend application with tailored dashboards for Patients, Doctors, and Admins. Integrate seamlessly with our FastAPI backend and prepare the entire stack for deployment.

## Wave 1: Frontend Foundation
1. **Shadcn UI Setup**: Install and configure `shadcn/ui` components (buttons, cards, forms, dialogs, tables).
2. **Global Styling**: Configure TailwindCSS with a premium dark-mode ready color palette, utilizing glassmorphism and micro-animations for a highly polished UX.
3. **API Client Setup**: Implement an Axios or Fetch client in `frontend/src/services/api.js` to manage JWT tokens (attaching them to headers) and handle automatic redirects on 401 Unauthorized errors.

## Wave 2: Role-Based Dashboards
1. **Patient Portal**:
   - `/patient/dashboard`: View upcoming appointments and historical visits.
   - `/patient/book`: The core booking flow (search doctors, select generated slots, input symptoms).
2. **Doctor Portal**:
   - `/doctor/dashboard`: View today's schedule and manage availability.
   - `/doctor/appointment/[id]`: Review Gemini's pre-visit AI summary, input post-visit clinical notes, and trigger the AI translation.
3. **Admin Portal**:
   - `/admin/doctors`: Add/edit doctor profiles, manage their working hours and leaves.

## Wave 3: Deployment
1. **Frontend**: Deploy Next.js to Vercel (connect GitHub repo).
2. **Backend**: Deploy FastAPI to Render using a `Dockerfile` and Gunicorn/Uvicorn.
3. **Database**: Point to a production Neon PostgreSQL instance.
