# Phase 2: Booking Engine & Integrations Plan

## Goal
Implement the transactional booking engine with row-locks, slot generation, and doctor leave management. Integrate external services including Gemini AI for medical summaries, Email (SMTP), Google Calendar, and background jobs.

## Wave 1: Booking Engine Core
1. **Slot Generation Service**: `backend/app/services/booking_service.py`
   - Algorithm to generate 30-minute intervals between a doctor's start and end times.
   - Filter out times that conflict with existing appointments or Doctor Leaves.
2. **Transactional Booking**: 
   - Use `SELECT ... FOR UPDATE` (or SQLAlchemy `with_for_update()`) on the time slot/doctor to prevent double booking.
   - Endpoint: `POST /api/appointments/book`.
3. **Leave Management**: 
   - Endpoints for Doctor/Admin to mark a day as 'Leave'.
   - Logic to automatically cancel and notify patients if they are booked on a newly marked leave day.

## Wave 2: AI Integrations
1. **Gemini Service**: `backend/app/services/llm_service.py`.
   - Setup `google-generativeai`.
   - Function 1: `analyze_symptoms(symptoms: str) -> (urgency, chief_complaint, questions)`. Triggered before visit.
   - Function 2: `translate_notes(notes: str) -> str`. Triggered after visit to create patient-friendly summary.
2. **Integrate into Booking**: Call the symptom analyzer when the patient books an appointment.

## Wave 3: Notifications & Background Jobs
1. **Email Service**: `backend/app/services/email_service.py`.
   - Generic async function to send HTML emails using `smtplib` (we will use dummy credentials or Mailtrap for dev).
2. **Calendar Service**: `backend/app/services/calendar_service.py`.
   - Functions to create `.ics` files and attach them to the emails (since setting up full Google OAuth flow for a 3-day sprint might block us).
3. **APScheduler Background Tasks**: `backend/app/workers/scheduler.py`.
   - Initialize APScheduler on app startup.
   - Job: Run every hour to find appointments starting in 24 hours and trigger email reminders.
