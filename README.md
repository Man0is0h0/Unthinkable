# Unthinkable Health

Unthinkable Health is a modern, AI-powered healthcare scheduling and consultation platform. It connects patients with doctors, streamlines appointment booking, and leverages AI (Groq API) to extract vital clinical summaries from raw patient symptoms before the doctor even begins the checkup. 

It is built with a **Next.js (React)** frontend, a **FastAPI (Python)** backend, and powered by **Neon Serverless Postgres**.

---

## 1. Local Setup Guide

### Prerequisites
- Node.js 18+ and npm
- Python 3.10+
- PostgreSQL database (or a free Neon Postgres instance)
- API Keys for Groq (LLM) and Brevo (Emailing)

### Frontend Setup
```bash
cd frontend
npm install
npm run dev
```
The frontend will start on `http://localhost:3000`.

### Backend Setup
```bash
cd backend
python -m venv venv
# On Windows:
venv\Scripts\activate
# On Mac/Linux:
# source venv/bin/activate

pip install -r requirements.txt
uvicorn app.main:app --reload
```
The backend API will start on `http://localhost:8000`. Swagger UI docs are available at `http://localhost:8000/docs`.

## 2. API Documentation

Our API is organized into several modules. Full OpenAPI documentation is automatically generated by FastAPI and available at `/docs`.

### Authentication (`/api/auth`)
- `POST /register`: Register a new patient or doctor.
- `POST /login`: Log in to receive an HttpOnly JWT cookie.
- `POST /logout`: Clear the authentication cookie.

### Profiles (`/api/profile`)
- `GET /me`: Get the current authenticated user's details and role.

### Bookings & Appointments (`/api/appointments`)
- `GET /slots/{doctor_id}`: Fetch available 30-min slots for a given date.
- `POST /book`: Book a new appointment and trigger AI symptom analysis and email confirmation.
- `GET /`: List appointments for a user.
- `POST /{id}/complete`: Complete a checkup, submit a prescription, and trigger AI to generate a patient-friendly summary.
- `POST /{id}/reschedule-tomorrow`: Reschedule a missed appointment.
- `POST /{id}/cancel`: Cancel an upcoming appointment.
- `POST /leave`: Doctors can mark themselves as on leave.

## 3. Database Schema

The database uses PostgreSQL with SQLAlchemy ORM.

### Key Tables
1. **users**: Base table for authentication.
   - `id`, `name`, `email`, `hashed_password`, `role` (patient, doctor, admin)
2. **patients**: Links to `users`.
   - `id`, `user_id`, `date_of_birth`, `phone`, `blood_group`
3. **doctors**: Links to `users`.
   - `id`, `user_id`, `specialty`, `qualifications`, `experience_years`, `availability_schedule`
4. **appointments**: Tracks visits.
   - `id`, `patient_id`, `doctor_id`, `date`, `start_time`, `end_time`, `status`, `symptoms`, `urgency`
   - `pre_visit_summary`: AI-generated clinical summary for the doctor.
   - `post_visit_summary`: AI-translated friendly summary for the patient.
   - `doctor_notes`: Raw medical notes.
5. **prescriptions**: Tracks medications issued during an appointment.
   - `id`, `appointment_id`, `medication_name`, `dosage`, `frequency`, `duration_days`
6. **doctor_leaves**: Tracks days doctors are unavailable.
   - `id`, `doctor_id`, `date`, `reason`

## 4. LLM Prompts & AI Integration

We use the **Groq API** (`llama3-8b-8192`) for ultra-fast medical text processing.

### Pre-Visit Triage (For Doctors)
When a patient books an appointment, they describe their symptoms. We prompt the LLM to act as a medical triage assistant:
> *"You are a medical triage assistant. Analyze the following patient symptoms. Extract the 'Chief Complaint', determine the 'Urgency' (Low, Medium, High), and suggest 3 relevant follow-up questions for the doctor to ask. Format as JSON."*

### Post-Visit Translation (For Patients)
After the doctor writes raw clinical notes (e.g., "Dx: Viral pharyngitis, DDx: Strep. Tx: supportive"), the LLM translates it into plain English for the patient dashboard:
> *"You are a friendly, empathetic medical assistant. Translate the following doctor's clinical notes into a simple, reassuring summary that the patient can easily understand without medical jargon."*

## 5. Google Calendar Setup Steps

The system automatically syncs appointments to the users' native calendar apps (including Google Calendar, Apple Calendar, and Outlook).

**How it works:**
1. When an appointment is booked, the backend generates an `.ics` (iCalendar) file using the `ics` Python library.
2. The `.ics` file is attached to the confirmation email and dispatched via the **Brevo API**.
3. **Google Calendar Integration:** When the patient opens the email in Gmail, Google automatically parses the attached `invite.ics` file and displays a rich "Add to Calendar" button at the top of the email. Clicking it syncs the appointment natively to their Google Calendar without requiring complex OAuth2 flows!
